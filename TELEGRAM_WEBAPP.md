# 🚀 TELEGRAM WEB APP - Инструкция по настройке

## Что это?

Telegram Web App позволяет открывать ваш веб-интерфейс **ПРЯМО ВНУТРИ Telegram**!
Пользователь нажимает кнопку в боте → открывается веб-страница в мессенджере.

---

## 📋 Шаги настройки:

### Шаг 1: Деплой сайта в интернет

Telegram Web App работает только с **HTTPS URL** (localhost не подойдет).

#### **Вариант A: ngrok (для теста)**

1. Скачай ngrok: https://ngrok.com/download
2. Запусти веб-сервер:
   ```bash
   cd backend
   python main.py
   ```
3. В новом терминале запусти ngrok:
   ```bash
   ngrok http 8000
   ```
4. Скопируй URL (например: `https://abc123.ngrok-free.app`)

#### **Вариант B: Railway.app (бесплатный хостинг)**

1. Зарегистрируйся: https://railway.app
2. Создай новый проект → Deploy from GitHub
3. Добавь переменные окружения:
   - `BOT_TOKEN=твой_токен`
   - `SECRET_KEY=любая_строка`
4. Railway даст URL типа: `https://your-app.railway.app`

#### **Вариант C: Render.com**

1. Зарегистрируйся: https://render.com
2. New Web Service → Connect repository
3. Build Command: `pip install -r requirements.txt`
4. Start Command: `uvicorn backend.main:app --host 0.0.0.0 --port $PORT`

---

### Шаг 2: Обновить URL в боте

Открой `backend/bot.py` и найди строку:

```python
WEB_APP_URL = "https://your-app-url.railway.app"  # <-- ИЗМЕНИ ЭТО!
```

**Замени на свой URL:**
```python
WEB_APP_URL = "https://abc123.ngrok-free.app"  # Если используешь ngrok
# ИЛИ
WEB_APP_URL = "https://your-app.railway.app"   # Если Railway
```

**ВАЖНО:** URL должен указывать на `/webapp`:
```python
WEB_APP_URL = "https://your-domain.com/webapp"
```

---

### Шаг 3: Перезапустить бота

```bash
cd backend
python bot.py
```

Должно появиться: `🤖 Бот запущен!`

---

### Шаг 4: Тестирование

1. Открой бота в Telegram
2. Напиши `/start`
3. Нажми кнопку **"🌐 Открыть веб-интерфейс"**
4. Веб-страница откроется **ВНУТРИ Telegram**!

## 🎯 Как это работает:

1. Пользователь нажимает кнопку в боте
2. Telegram открывает URL: `https://your-domain.com/webapp`
3. Страница получает `telegram_id` из Telegram API
4. Запросы идут к специальным эндпоинтам (без JWT):
   - `GET /api/telegram/tasks/{telegram_id}` - получить задачи
   - `POST /api/telegram/tasks` - создать задачу
   - `PATCH /api/telegram/tasks/{id}/complete` - выполнить

## 🐛 Возможные проблемы:

### Ошибка: "This bot can't open Web Apps"

**Решение:** URL должен быть HTTPS (не HTTP!)

### Веб-страница не загружается

**Проверь:**
1. Сервер запущен и доступен по URL
2. URL в bot.py правильный
3. В боте указан URL на `/webapp` endpoint

### Задачи не загружаются

**Проверь в браузере:**
1. Открой: `https://your-domain.com/api/telegram/tasks/123` (где 123 - твой telegram_id)
2. Должен вернуть JSON с задачами
